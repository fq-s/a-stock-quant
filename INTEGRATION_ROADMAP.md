# A股量化项目整合路线

## 当前主底座

本项目以 `linhuaxiamen/a-stock-quant` 为主底座，优先保留已经具备的能力：

- AKShare A股数据获取和本地缓存
- 单股与组合回测
- 因子库、多因子选股、指数股票池
- 回测和模拟盘共用的风险控制
- 模拟盘与 QMT 接口
- Web 管理面板
- 自动化测试

## 第一阶段：先稳定跑通

目标是先形成一个可以重复验证的研究闭环，不连接真实账户。

1. 安装 Python 3.11 或 3.12。
2. 安装 `requirements.txt` 和 `requirements-dev.txt`。
3. 运行全部测试。
4. 运行单股回测和组合回测。
5. 启动 Web 面板查看结果。
6. 使用模拟盘验证信号、风控和通知。

验收标准：

- 全部测试通过。
- 同一组参数重复运行，结果一致。
- 风控触发记录清楚。
- 模拟盘不产生真实订单。

## 第二阶段：借鉴 china-astock-quant

参考项目：

- https://github.com/MisakaMikoto128/china-astock-quant

优先借鉴：

- `BaseBroker` / `PaperBroker` / `QMTBroker` 的统一券商接口
- T+1、100股整数手、佣金、印花税、过户费、滑点规则
- 参数网格搜索与 walk-forward 滚动验证
- 回测与模拟盘一致性测试

注意：

- 不直接复制对方的回测收益结论。
- 每个交易规则都要增加本项目自己的测试。
- 真实账户接入前，必须先通过模拟盘。

## 第三阶段：接入 AKQuant 能力

参考项目：

- https://github.com/akfamily/akquant

优先借鉴：

- 交互式 HTML 回测报告
- 基准、超额收益、回撤和归因分析
- 参数搜索和 walk-forward 验证
- 更完整的技术指标与因子计算

暂不迁移：

- Rust 核心引擎
- 复杂订单和机器学习模块

原因是当前阶段应优先保证项目简单、可理解、可验证。

## 第四阶段：参考 OSkhQuant 改进界面

参考项目：

- https://github.com/khscience/OSkhQuant

优先借鉴：

- Windows 本地操作体验
- 股票池管理
- 策略参数配置
- 回测结果展示
- MiniQMT 工作流

本项目优先继续使用现有 Web 面板，不急于改为 PyQt 桌面程序。

## 实盘前硬性条件

- 模拟盘连续运行并记录结果。
- 单股最大仓位、总仓位、最大回撤、单日亏损规则已启用。
- 人工确认账户、路径和交易权限。
- 默认关闭真实下单。
- 真实下单必须有单独开关和醒目提示。
- 首次实盘只使用可承受损失的小额资金。

## 建议开发顺序

1. 修复并跑通现有测试。
2. 增加 A股交易规则测试。
3. 增加策略参数稳定性报告。
4. 完善 Web 面板。
5. 长时间运行模拟盘。
6. 最后才接入 QMT 真实账户。
